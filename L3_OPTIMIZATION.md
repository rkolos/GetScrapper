# L3 Оптимизация: Stateless Browserbase Scraping

## Обзор

Реализована оптимизированная функция `fetch_via_browserbase` для Уровня 3 (L3) эскалации, которая использует **бессессионный** эндпоинт `/v1/scrape` Browserbase API для получения только текстового контента.

## Ключевые преимущества

### 1. Stateless подход
- **Один POST-запрос** вместо множественных запросов для создания/управления сессией
- **Автоматическое управление жизненным циклом** браузера на стороне Browserbase
- **Нет риска "оставшихся сессий"** - браузер уничтожается автоматически

### 2. Оптимизация ресурсов
```json
{
  "loadOptions": {
    "blockResources": ["image", "font", "stylesheet", "media"]
  }
}
```
- Блокирует загрузку изображений, шрифтов, CSS и медиа
- **Экономия трафика** и **ускорение загрузки**
- **Снижение стоимости** использования API

### 3. Целевое извлечение текста
```json
{
  "jsScript": "document.body.innerText"
}
```
- Выполняет JavaScript на сервере Browserbase **после** рендеринга
- Возвращает **чистый текст** без HTML-разметки
- **Обходит JS-защиту** и динамический контент

## Использование

### 1. Установка переменной окружения
```bash
export BROWSERBASE_API_KEY="your_api_key_here"
```

### 2. Импорт и вызов
```python
from getscrapper.core.fetchers.browserbase_fetcher import fetch_via_browserbase

# Получение текста страницы
text_content = fetch_via_browserbase("https://example.com")

if text_content:
    print(f"Получено {len(text_content)} символов")
    print(text_content[:200])  # Первые 200 символов
else:
    print("Ошибка получения контента")
```

## Обработка ошибок

Функция обрабатывает следующие типы ошибок:

1. **Отсутствие API ключа**
   ```
   [L3 FATAL] BROWSERBASE_API_KEY не установлен.
   ```

2. **HTTP ошибки API** (401, 403, 429, 500)
   ```
   [L3 FATAL] Browserbase API HTTP Error: 401 | Response: {"error": "Unauthorized"}
   ```

3. **Сетевые ошибки** (Timeout, DNS)
   ```
   [L3 FATAL] Browserbase API Network Error: Connection timeout
   ```

## Конфигурация

### Таймаут
- **90 секунд** - достаточно для сложных JS-приложений
- Настраивается в параметре `timeout=90`

### Блокировка ресурсов
- `image` - изображения
- `font` - шрифты  
- `stylesheet` - CSS файлы
- `media` - видео/аудио

## Сравнение с предыдущей реализацией

| Аспект | Старая реализация | Новая L3 реализация |
|--------|------------------|-------------------|
| **Запросы** | 4+ запроса (создание сессии, навигация, получение контента, закрытие) | 1 запрос |
| **Управление сессией** | Ручное создание/закрытие | Автоматическое |
| **Риск утечек** | Высокий (забытые сессии) | Отсутствует |
| **Оптимизация** | Загружает все ресурсы | Блокирует ненужные |
| **Результат** | HTML + метаданные | Чистый текст |
| **Производительность** | Медленная | Быстрая |
| **Стоимость** | Высокая | Низкая |

## Интеграция в систему

L3 функция интегрирована в существующую архитектуру:

1. **`BrowserbaseFetcher`** - основной класс для асинхронных операций
2. **`fetch_via_browserbase`** - синхронная L3 функция для простых случаев
3. **Обратная совместимость** - старые методы остаются доступными

## Заключение

Новая L3 реализация обеспечивает:
- ✅ **Максимальную эффективность** - один запрос вместо множественных
- ✅ **Надежность** - нет риска утечек сессий  
- ✅ **Экономичность** - блокировка ненужных ресурсов
- ✅ **Простота** - stateless подход без управления жизненным циклом
- ✅ **Целевое извлечение** - только нужный текстовый контент
